Understanding CSS Grid: A Complete Guide to Two-Dimensional Layouts

CSS Grid changed how we build web layouts. This guide takes you from the basics of rows and columns to complex named template areas that make your HTML self-documenting.

BV
Benjamin Vanderzeypen Instructor at HackYourFuture Belgium

Before CSS Grid, building a two-dimensional layout meant nested flexbox containers, float hacks, or reaching for a heavy framework. Grid changed all of that.

What makes Grid different from Flexbox?

Flexbox is one-dimensional — it arranges items in a row or a column. Grid is two-dimensional: it handles rows and columns simultaneously. The key mental model shift is this: with Flexbox, you start from the content and let it dictate the layout. With Grid, you define the layout first and place content into it.

"Grid is for layout. Flexbox is for components." While this is an oversimplification, it is a useful starting point.

— Jen Simmons, Mozilla

Your first grid

To create a grid, you only need two things: a container with display: grid and some children. But a grid with no defined tracks is just a single column — not very interesting. The real power comes from grid-template-columns.

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1.5rem;
}

The repeat(3, 1fr) creates three equal columns. The fr unit represents a fraction of the available space — it is like flex: 1 for Grid.

Named template areas

This is where Grid becomes truly powerful. Instead of thinking in abstract row and column numbers, you can name areas of your grid like a map:

.page {
  display: grid;
  grid-template-areas:
    "header  header"
    "sidebar content"
    "footer  footer";
  grid-template-columns: 280px 1fr;
  grid-template-rows: auto 1fr auto;
}

.page-header { grid-area: header; }
.page-sidebar { grid-area: sidebar; }
.page-content { grid-area: content; }
.page-footer  { grid-area: footer; }

The visual representation in the CSS maps directly to the visual layout in the browser. Your HTML structure and your CSS tell the same story.

Conclusion

CSS Grid is not a replacement for Flexbox — they are complementary tools. Use Grid when you need a two-dimensional layout with defined tracks. Use Flexbox when you need one-dimensional flow, like a navigation bar or a list of tags.
